# 乙游主题 Methods Review 下一步

## 审查定位

当前题目可以暂定为：**情绪价值的商业化：女性向游戏中的数字亲密关系、品牌依恋与持续消费**。

按 `methods-reviewer` 的思路，下一步不是继续扩充文献，而是先把研究设计审清楚：研究对象是谁、核心暴露是什么、结果变量是什么、可支持什么强度的结论，以及哪些地方不能过度宣称。

## 建议设计路线

优先采用混合方法，而不是一开始就只做问卷：

1. 质性探索：访谈或 netnography，用来确认乙游玩家如何描述数字亲密、情绪价值、付费合理化和角色依恋。
2. 问卷模型：用 SEM/PLS-SEM 检验“数字亲密关系 -> 情绪价值 -> 品牌依恋 -> 持续消费意向/行为”的关联链条。
3. 情境实验：用 AI披露、互动亲密度、限时付费压力或角色回应方式做操纵，检验短期心理机制。
4. 可选文本分析：用评论、社群帖、游戏评价抽取情绪价值和亲密叙事，但必须有人工验证样本。

## 主路径建议

主路径只保留一条，避免模型太散：

`角色互动/数字亲密关系 -> 情绪价值 -> 品牌依恋 -> 持续消费`

可选调节变量：

- AI披露或角色真实感
- 玩家孤独感/情感需求
- 付费能力或重度玩家身份
- 社群归属

## 立即可执行任务

1. 写 `study_design_brief.md`：一句研究问题、研究对象、单位、暴露、机制、结果、可支持的结论类型。
2. 写 `construct_ledger.csv`：每个核心构念的定义、量表来源、题项方向、相邻构念区别。
3. 写 `data_plan.md`：问卷样本、访谈样本、文本语料、行为数据能否获取。
4. 做一个最小问卷草案：每个构念 3-5 个题项，先用于预测试，不急着正式发放。
5. 做一个 2x2 实验草案：例如 AI披露(有/无) x 互动亲密度(高/低)。

## 作者需要先决定

- 研究对象到底是“乙游玩家”，还是更宽的“虚拟亲密消费用户”？
- 结果变量要主打“持续消费意向”，还是尽量收集真实/自报付费行为？
- 研究定位是解释机制，还是评估平台商业化/操纵风险？
- AI男主/AI陪伴是否是核心对象，还是作为近年扩展背景？

# Study Design Brief: Cluster C

## Route

route_id: `cluster_c_digital_self_virtual_goods`

working title: `从虚拟物品到数字自我：乙游角色资产、情绪价值与持续消费`

selected route: Cluster C，数字自我、虚拟商品与游戏消费。

rejected alternatives: 暂不以 AI 陪伴或虚拟影响者为主线。它们可作为近年背景和机制类比，但不作为核心研究对象。

## Model

Population: 近 3 或 6 个月玩过国内女性向恋爱手游的成年玩家。

Unit of analysis: 首选 `玩家-最喜欢角色关系`。备选为 `玩家`。

Setting: 国内乙游场景，重点是抽卡、角色收藏、剧情记忆、账号资产、角色养成和二创/社群实践。

Core constructs:

- 数字自我延伸：玩家是否把角色、卡牌、账号资产、剧情记忆视为自我表达或身份的一部分。
- 虚拟物品情感价值：虚拟角色/卡牌/收藏物带来的陪伴、愉悦、安慰、审美和纪念意义。
- 角色依恋：玩家对特定角色形成的情感纽带。
- 品牌/IP依恋：玩家对游戏品牌、世界观或 IP 的情感纽带。
- 持续消费：继续游玩、继续付费、抽卡/月卡/周边/活动消费意向或行为。

Mechanism candidate:

`虚拟物品/角色资产的数字自我延伸 -> 情绪价值 -> 角色依恋/品牌依恋 -> 持续消费`

Scope conditions:

- 适用于有角色偏好、抽卡/收藏/养成经历的乙游玩家。
- 不直接外推到所有游戏玩家、非恋爱向手游玩家或未成年人。
- 如果只有横截面问卷，研究只能支持关联和机制一致性，不能支持强因果结论。

## Inquiry

Primary inquiry:

在国内乙游玩家中，玩家对虚拟角色资产的数字自我延伸感是否与更高的情绪价值、角色/品牌依恋和持续消费意向相关？

Target comparison:

数字自我延伸感较高的玩家-角色关系，与数字自我延伸感较低的玩家-角色关系相比，是否表现出更高的情绪价值、角色依恋、品牌依恋和持续消费意向？

Outcome:

主结果为持续消费意向。若可获得行为数据，则加入近 30 天或近 3 个月自报付费金额、抽卡次数、月卡/通行证购买、活动参与等行为指标。

Exposure:

玩家对角色/卡牌/账号资产/剧情记忆的数字自我延伸感。

Interpretation:

横截面问卷版本：关联性和理论机制一致性。

实验或纵向版本：可检验短期机制或时间顺序，但仍需谨慎处理平台选择和玩家自选择。

## Data Strategy

Minimum data:

- 问卷样本：国内乙游成年玩家。
- 纳入标准：近 3 或 6 个月玩过至少一款乙游；能指出一个最喜欢或投入最多的角色；同意报告基本游玩和消费信息。
- 推荐样本量：预测试 50-80；正式问卷至少 300，若 SEM 模型复杂则争取 500 以上。

Optional qualitative data:

- 访谈 15-25 人，覆盖高付费、低付费、剧情党、抽卡党、二创/社群参与者。
- 用于修订数字自我延伸和虚拟物品情感价值的题项。

Optional text data:

- TapTap/应用商店评论、微博/小红书公开帖、玩家论坛或超话。
- 只能作为辅助证据，除非建立人工验证样本、codebook 和一致性指标。

Missingness and permissions:

- 不收集未成年人数据，或单独设计伦理审批。
- 自报消费可使用区间题，降低敏感性。
- 若要求截图验证，需要单独说明隐私处理方式。

## Answer Strategy

First quantitative answer:

- 描述统计：样本画像、玩龄、付费经历、主推角色类型、收藏/抽卡行为。
- 测量模型：CFA 或 PLS measurement model，检查 CR、AVE、HTMT、载荷。
- 主模型：SEM/PLS-SEM 检验数字自我延伸、情绪价值、角色依恋/品牌依恋与持续消费意向之间的路径。
- 稳健性：区分付费玩家和非付费玩家；区分高频玩家和低频玩家；替换结果变量为自报付费行为。

First qualitative answer:

- 主题编码：角色资产如何被玩家描述为“我的”“纪念”“陪伴”“证明投入”“身份标签”。
- 负例：有收藏但无强依恋、有依恋但不付费、反感商业化但持续游玩。

## Diagnose

Key diagnosands:

- 判别效度：数字自我延伸、情绪价值、角色依恋、品牌依恋是否可区分。
- 共同方法偏差：同源问卷、同一时间测量导致路径虚高。
- 样本偏差：社群招募会高估重度玩家和高情感投入者。
- 结果行为化：持续消费意向不能等同于真实持续消费。
- 因果语言：横截面 SEM 不支持“导致”。

## Redesign

If construct overlap is too high:

- 合并角色依恋和品牌依恋，或将品牌依恋作为次级结果。
- 将数字自我延伸拆为身份表达、记忆保存、投入证明三个维度。

If sample is too biased:

- 分渠道报告结果。
- 将结论限定为社群活跃乙游玩家。

If behavior data unavailable:

- 主结果改为持续消费意向和自报消费区间。
- 在报告边界中明确不能证明实际留存或长期付费。

## Report Boundary

This design can support:

- 乙游虚拟物品和角色资产与情绪价值、依恋、持续消费意向之间的关联。
- 数字自我延伸作为解释乙游情感消费的一条机制路径。
- 对虚拟商品和角色资产商业化的理论解释。

This design cannot support without stronger data:

- 数字自我延伸导致持续消费。
- 所有乙游玩家都将角色视为自我延伸。
- 平台商业化策略一定构成情感操纵。

## First Analysis Plan

Smallest first loop:

1. 做 15 人访谈或 50 人预测试。
2. 检查 Cluster C 核心构念题项是否能被玩家理解。
3. 输出一张构念相关矩阵和一张 HTMT/AVE/CR 表。
4. 决定是否保留数字自我延伸、情绪价值、角色依恋、品牌依恋四个独立构念。

## Handoff

next_skill_route:

- `research-data-builder`：构建问卷数据字典、编码规则和样本流。
- `research-analysis-runner`：有预测试数据后跑描述统计、CFA/PLS-SEM 初版。
- `methods-reviewer`：预测试后审查测量重叠、样本偏差和因果语言。

Author decisions required:

- 分析单位选玩家还是玩家-角色关系。
- 品牌依恋对象是角色、游戏品牌还是 IP。
- 是否收集真实/截图验证消费行为。
- 是否加入伦理感知或商业化反感作为边界变量。
